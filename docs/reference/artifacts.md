# Artifacts and Data Layout

This page lists the important input and output contracts by stage.

## Source Data

```text
data/KITTI-360/
  calibration/perspective.txt
  data_poses/<drive>/cam0_to_world.txt
  images/<drive>/image_00/data_rect/*.png
  images/<drive>/image_01/data_rect/*.png
```

## Prepared Octree-AnyGS Data

Stage: `prepare`  
Default path: `/data/COLMAP/<drive>/`

```text
/data/COLMAP/<drive>/
  images/
    *.png
  sparse/0/
    cameras.txt
    images.txt
    points3D.ply
  metadata.json
```

## Octree-AnyGS Training Runs

Stage: `train`  
Default path: `/data/OCTREE-ANYGS/<drive>/<timestamp>/`

Important files:

```text
config.yaml
input.ply
point_cloud/iteration_<N>/point_cloud_anchor.ply
```

`bucket` defaults to the latest run directory under `/data/OCTREE-ANYGS/<drive>`.
Use `--model-path` to pin a specific run.

## Stereo Point Cloud

Stage: `stereo`  
Default path: `data/points_world/<drive>/`

```text
points_world.npz
points_world.ply
metadata.json
```

`points_world.npz` contains:

| Key | Meaning |
| --- | --- |
| `xyz` | World-frame positions |
| `rgb` | RGB values from the left image |
| `frame_id` | Source KITTI-360 frame id |

## Bucketing and Normalization

Stage: `bucket`  
Default path: `data/m4/<drive>/`

```text
points_norm.npz
pts_by_anchor.npz
norm_params.json
bucket_metadata.json
```

`points_norm.npz` stores normalized `xyz + rgb` rows used by VBGS.  
`pts_by_anchor.npz` stores packed per-anchor point indices.  
`norm_params.json` stores the global offset and standard deviation.

## Anchor Posterior Fits

Stage: `fit`  
Default path: `data/m4/<drive>/`

```text
anchor_posterior.npz
fit_metadata.json
```

Important posterior fields include:

| Field | Use |
| --- | --- |
| `alpha` | Dirichlet mixture parameters |
| `final_k` | Accepted component count per anchor |
| `is_observed` | Observed-anchor mask |
| `spatial_kappa`, `spatial_u`, `spatial_n` | Normal-Wishart spatial parameters |
| `delta_kappa`, `delta_u`, `delta_n` | Color/delta posterior parameters |

## Scalar Uncertainty

Stage: `uncertainty`  
Default path: `data/m4/<drive>/`

```text
U.npy
uncertainty_components.npz
uncertainty_metadata.json
uncertainty_histogram.png
```

`U.npy` is a one-dimensional float array with shape `[N_anchors]`.

## Map and View Diagnostics

Stages: `map-viz`, `render`, `nbv`, `nbv-viz`  
Default curated path: `outputs/v1_0/<drive>/`

```text
pointclouds/
  stereo/
  anchors/
views/
nbv/
uncertainty/
prepared/
octree/
run_manifest.json
```

The final bundle is:

```text
outputs/v1_0/<drive>.zip
```

## Generated Configs

Octree-AnyGS configs generated by the training wrapper are written under:

```text
generated_configs/
```

These are useful for reproducing training runs and checking exactly which
Octree-AnyGS parameters were passed.
